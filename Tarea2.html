Evidencia M2

  Instrucciones de la actividad

Aplicación web (front-end y back-end) que permita enviar los datos de un formulario de contacto a la dirección de correo electrónico del profesor. Durante las pruebas, se sugiere que usen su propio correo como destino.

Se deberá entregar un archivo zip con los archivos de la aplicación y sus instrucciones. Los archivos deben contener el código fuente completo de la aplicación y las instrucciones deberán especificar los pasos necesarios para compilarlo, configurarlo (cambiar el correo de destino) y ejecutarlo. Los lenguajes de back-end permitidos son .NET, PHP, Ruby on Rails, Node.js o Aplicaciones EE de Java que puedan ser ejecutadas con Tomcat o Glassfish.

 

La aplicación deberá cumplir con los siguientes criterios requeridos:

- El formulario deberá solicitar los siguientes datos: Nombre completo, correo electrónico, fecha de nacimiento, Tema, comentario.

- El nombre completo deberá de tener al menos 3 palabra.

- El correo electrónico deberá tener un formato válido (por ejemplo persona@correo.com).

- La fecha de nacimiento deberá presentar un calendario para seleccionarla. La fecha deberá ser en formato dd/mm/aaaa y el usuario no podrá ser menor a 15 años ni mayor a 120.

- El tema deberá seleccionarse de una lista donde las opciones deberán cargarse por Ajax desde la siguiente ruta: http://cursos.ectotec.com/javascript/api/temas. Se deberá validar que el tema seleccionado exista en esa ruta.

- El comentario es un texto abierto que no debe exceder 1000 caracteres.

- Deberá contar con un botón de enviar y uno de limpiar. El de limpiar deberá borrar el formulario. El de enviar, deberá preguntar al usuario si está seguro, y de ser así, enviar el formulario.

- Cuando el formulario sea enviado, se deberá presentar una pantalla al usuario indicándole que sus datos fueron recibidos.

- El correo con los datos deberá incluir todos los campos en un diseño tipo tabla. Adicionalmente deberá capturar la IP del usuario emisor, así como la fecha y hora de envío.

  Forma de entrega

 Individual, archivo zip en la plataforma
